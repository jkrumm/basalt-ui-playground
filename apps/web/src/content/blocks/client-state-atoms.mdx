## Overview

Two Jotai primitives handle all client-side state in this project:

- `atom()` — session-only, cleared when the tab closes (modals, command palette, popovers)
- `atomWithStorage()` — persists to localStorage (view mode, sort order, theme preference)

All atoms are defined in `src/atoms/` — ESLint enforces this location with a lint error
if you call `atom()` or `atomWithStorage()` outside that directory.

## Atom Definitions

```typescript filename="apps/web/src/atoms/ui.atoms.ts"
import type { UserPreferences } from "@basalt-ui-playground/schemas";
import { atom } from "jotai";
import { atomWithStorage } from "jotai/utils";

// Session-only — cleared when the browser tab closes
export const searchOpenAtom = atom<boolean>(false);

// Persistent — survives page refresh via localStorage
export const viewModeAtom = atomWithStorage<UserPreferences["viewMode"]>(
  "basalt-view-mode", // localStorage key — must be unique across all atoms
  "grid", // default value on first visit
);

export const sortByAtom = atomWithStorage<UserPreferences["sortBy"]>("basalt-sort-by", "default");
```

```typescript filename="apps/web/src/atoms/index.ts"
// Barrel re-export — always import from '~/atoms', not from individual files
export * from "./preferences.atoms";
export * from "./ui.atoms";
```

## Reading and Writing Atoms

Three hooks cover all access patterns:

```tsx
import { useAtom, useAtomValue, useSetAtom } from "jotai";
import { searchOpenAtom, viewModeAtom } from "~/atoms";

// useSetAtom — write-only, no re-render when the atom value changes
function NavSearchButton() {
  const setSearchOpen = useSetAtom(searchOpenAtom);
  return <Button onClick={() => setSearchOpen(true)}>Search</Button>;
}

// useAtom — read + write, re-renders on every change
function ViewToggle() {
  const [viewMode, setViewMode] = useAtom(viewModeAtom);
  return (
    <Button onClick={() => setViewMode(viewMode === "grid" ? "list" : "grid")}>
      {viewMode === "grid" ? "Table view" : "Grid view"}
    </Button>
  );
}

// useAtomValue — read-only, re-renders on every change
function ViewLabel() {
  const viewMode = useAtomValue(viewModeAtom);
  return <span>Showing as {viewMode}</span>;
}
```

Prefer `useSetAtom` in write-only components (buttons, action handlers) — it avoids
subscribing to the atom value and prevents unnecessary re-renders.

## Server-Sync Override

When the user visits `/settings`, server preferences override the localStorage atoms.
The `useEffect` dependency array ensures this runs when `serverPrefs` changes (e.g. after
a successful save), not on every render.

```tsx filename="apps/web/src/routes/_protected/settings.tsx"
function SettingsForm() {
  const { data: serverPrefs } = useSuspenseQuery(userPreferencesQuery());
  const setViewMode = useSetAtom(viewModeAtom);
  const setSortBy = useSetAtom(sortByAtom);

  // Server-authoritative override — local state converges with server on settings visit.
  // useSetAtom references are stable so this won't re-run unless serverPrefs actually changes.
  useEffect(() => {
    setViewMode(serverPrefs.viewMode);
    setSortBy(serverPrefs.sortBy);
  }, [serverPrefs, setViewMode, setSortBy]);
}
```

## ESLint Enforcement Rules

Three rules enforce the atom conventions across the whole codebase:

```javascript filename="apps/web/eslint.config.mjs"
// 1. Atom creation must happen in src/atoms/ — enforced on all src/**/*.{ts,tsx}
{
  selector: 'CallExpression[callee.name=/^atom/]',
  message: 'Jotai atoms must be defined in src/atoms/, not inline in components.',
}

// 2. Atom variables must use the *Atom suffix
{
  selector: "VariableDeclarator[init.callee.name=/^atom/]:not([id.name=/.+Atom$/])",
  message: 'Jotai atoms must use the *Atom naming convention (e.g., viewModeAtom).',
}

// 3. atomFamily is banned — causes infinite re-renders with React Compiler
{
  selector: "CallExpression[callee.name='atomFamily']",
  message: 'atomFamily is broken with React Compiler. Use static atoms instead.',
}
```

<Admonition type="warning" title="atomFamily is banned">
  `atomFamily` from `jotai/utils` causes an infinite re-render loop when React Compiler is active.
  Use static atoms with typed parameters instead. If you need per-key atom instances, use a `Map`
  and manage lifecycle manually.
</Admonition>

## Jotai DevTools

In development, the Jotai DevTools panel shows all active atoms and their current values
in real time. It is mounted unconditionally in `__root.tsx` but guarded to DEV only:

```tsx filename="apps/web/src/routes/__root.tsx"
import { DevTools } from "jotai-devtools";
import "jotai-devtools/styles.css";

// Inside the Jotai Provider, DEV only:
{
  import.meta.env.DEV && <DevTools />;
}
```

The DevTools panel appears as a floating button in the bottom-left corner of the screen
during development. Click it to inspect any atom's current value and subscription count.
