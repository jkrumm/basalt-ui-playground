## Overview

The boilerplate ships with a complete mobile baseline — touch behavior, iOS zoom prevention,
and the web app manifest structure. Most of this is **pre-configured and requires no changes**
per app.

The one task you must complete per app is providing the icon assets. Everything else in
`__root.tsx`, `app.css`, and the manifest is ready to use.

---

## What is pre-configured

These are set once in the boilerplate and require no per-app work:

**`app.css` — mobile & touch baseline**
- `-webkit-text-size-adjust: 100%` — prevents font inflation on iOS rotation
- `overscroll-behavior-y: none` — disables pull-to-refresh in the app shell
- `-webkit-tap-highlight-color: transparent` — removes the blue tap flash on iOS
- `-webkit-touch-callout: none` — disables long-press action sheets on images and links
- Blueprint input font-size override (`max(16px, 1em)`) — prevents iOS zoom on input focus

**`__root.tsx` — head meta**
- `theme-color` (light + dark via `prefers-color-scheme`) — tints the browser chrome on Android
- `apple-mobile-web-app-capable` + `apple-mobile-web-app-status-bar-style` — enables standalone
  mode when added to the iOS home screen
- `<link rel="manifest">` — points to `public/manifest.webmanifest`

**`public/manifest.webmanifest`** — `display: standalone`, theme colors, icon slots defined

**Stylelint** — `unit-disallowed-list: ["vh"]` enforces `dvh` over `vh` across all CSS

## Per-app: icon assets

The manifest references three icon files that you must create for each app:

| File | Size | Purpose |
|------|------|---------|
| `public/icon-192.png` | 192×192 | Android home screen, PWA splash |
| `public/icon-512.png` | 512×512 | Android launcher, maskable |
| `public/apple-touch-icon.png` | 180×180 | iOS "Add to Home Screen" |

### Generating icons

**[realfavicongenerator.net](https://realfavicongenerator.net)** — upload one high-res PNG or SVG,
download all sizes pre-named. Also generates the HTML snippet and validates your live setup.

**[pwa-asset-generator](https://github.com/elegantapp/pwa-asset-generator)** — CLI tool, best fit
for this stack. One command from your source SVG:

```bash
bunx pwa-asset-generator logo.svg public/ --manifest public/manifest.webmanifest
```

Generates all sizes, writes them to `public/`, and patches the manifest `icons` array automatically.

**[maskable.app](https://maskable.app)** — after generating the 512px icon, paste it here to
preview how Android crops it into a circle or squircle. The icon needs meaningful content inside
the center 80% "safe zone" to avoid unwanted cropping.

### Icon requirements

- **No transparency** on the `purpose: any maskable` icon — Android fills transparent areas
  unpredictably. Use a solid background matching your `background_color` in the manifest.
- **Solid color background** on `apple-touch-icon.png` — iOS renders it with rounded corners;
  transparent PNGs show as black.
- Source file should be at least **512×512px** before scaling down.

## Per-app: manifest customization

Update `public/manifest.webmanifest` with the app's identity:

```json
{
  "name": "Your App Name",
  "short_name": "App",
  "description": "One-sentence description",
  "theme_color": "#1c2127",
  "background_color": "#1c2127"
}
```

- `short_name` appears under the icon on the home screen — keep it under 12 characters
- `theme_color` and `background_color` should match the app's primary background
- Update the matching `theme-color` meta values in `__root.tsx` to stay in sync

## Not yet implemented: safe area insets

`viewport-fit=cover` is intentionally not set. Adding it causes iOS to extend content under
the status bar, which requires `env(safe-area-inset-top)` handling on any fixed top element
(e.g. the nav bar) and `env(safe-area-inset-bottom)` on any fixed bottom element.

Add this when building a proper mobile navigation shell:

**`__root.tsx`** — update viewport meta:
```ts
{ name: 'viewport', content: 'width=device-width, initial-scale=1, viewport-fit=cover' }
```

**Fixed top element (e.g. nav):**
```css
padding-top: env(safe-area-inset-top);
```

**Fixed bottom element (e.g. mobile tab bar):**
```css
padding-bottom: env(safe-area-inset-bottom);
```

Both `env()` values resolve to `0px` on non-notch devices and desktop — safe to apply globally
on any fixed shell element.

## OG / social images

Static OG images should be 1200×630px. Per-route `og:image` tags are already wired into
`blog/$slug.tsx` — reference a static asset in `public/` or an absolute URL.

For dynamic OG images generated from content (e.g. per-post cards with title and author),
[Satori](https://github.com/vercel/satori) renders JSX to PNG at build or request time
without a browser dependency. Works in Bun/Node.

See the [SEO doc](/docs/seo) for the full per-route meta pattern.

## Checklist — per app

- [ ] Generate `icon-192.png`, `icon-512.png`, `apple-touch-icon.png` and place in `public/`
- [ ] Check `icon-512.png` in [maskable.app](https://maskable.app) — content visible in safe zone
- [ ] Update `manifest.webmanifest` — `name`, `short_name`, `description`, colors
- [ ] Update `theme-color` meta values in `__root.tsx` to match the manifest colors
- [ ] Add a favicon (`public/favicon.ico` or `<link rel="icon">` in `__root.tsx`)
- [ ] Validate with [realfavicongenerator.net/favicon_checker](https://realfavicongenerator.net/favicon_checker) after deploy
