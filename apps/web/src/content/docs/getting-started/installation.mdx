## Requirements

| Tool    | Minimum version |
| ------- | --------------- |
| Node.js | 24.0.0          |
| pnpm    | 10.0.0          |

## Environment

The project requires no environment variables for local development.
The CBBI API data is fetched from a public endpoint at runtime.

## Project structure

```
apps/
  web/           # TanStack Start frontend
    content-collections.ts   # MDX pipeline config (Zod schemas, compileMDX)
    src/
      components/   # Blueprint + MDX components
      content/      # MDX blog posts, docs, guides, blocks
      lib/          # Shared utilities (content.ts, sitemap.ts, ...)
      routes/       # TanStack Router file-based routes
      styles/       # Global CSS
  api/           # Hono API server

packages/
  schemas/       # Shared Zod schemas (workspace package)
```

## Available scripts

Run from `apps/web/`:

| Script               | Description                        |
| -------------------- | ---------------------------------- |
| `pnpm run dev`       | Start Vite dev server on port 3000 |
| `pnpm run build`     | Production build to `dist/`        |
| `pnpm run start`     | Start the production server        |
| `pnpm run typecheck` | TypeScript type checking           |

Run from the workspace root:

| Script     | Description                         |
| ---------- | ----------------------------------- |
| `vp lint`  | ESLint across all packages          |
| `vp fmt`   | Format all files with Oxc formatter |
| `vp build` | Build all apps                      |

<Admonition type="note">
  `vp` is the `vite-plus` unified toolchain CLI — it wraps lint, format, build, and dev server in a
  single command. Run `vp --help` to see all options.
</Admonition>
