## Overview

This post demonstrates all available MDX features in this content system.
The pipeline uses `@mdx-js/rollup` with Shiki for compile-time syntax highlighting — zero runtime JS for code blocks.

<Admonition type="tip" title="Before you start">
  Make sure you have Bun ≥ 1.2 installed. The examples below use `bun` as the package manager.
</Admonition>

## Installation

<PackageManagerTabs command="add @mdx-js/rollup rehype-shiki" />

## Configuration Steps

<Steps>
  <Step title="Install dependencies">
    Add the MDX rollup plugin and remark/rehype plugins to your project.
  </Step>
  <Step title="Configure vite.config.ts">
    Add the MDX plugin before `tanstackStart()`. The `enforce: 'pre'` is critical — MDX must transform files before Vite sees them as JSX.

    ```typescript
    { enforce: 'pre', ...mdx({ remarkPlugins: [...], rehypePlugins: [...] }) }
    ```
  </Step>
  <Step title="Create content files">
    Add `.mdx` files to `src/content/blog/` or `src/content/docs/`. Frontmatter is automatically exported as a named `frontmatter` constant.
  </Step>
</Steps>
